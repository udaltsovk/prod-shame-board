# 📛 Как добавить запись на доску позора

## Быстрый способ — через Issue-форму

1. Открой [**New Issue**](../../issues/new?template=shame.yml)
2. Заполни форму
3. Мейнтейнер ставит лейбл `approved` → бот создаёт PR → запись на сайте

## Как вставлять цитаты

### Одна фраза

Просто пиши текст:

```
GIL не влияет на производительность
```

### Несколько сообщений одного человека

Каждое сообщение с новой строки:

```
так нельзя вызывать асинхронную функцию
ну то есть технически можно
но это не будет работать как надо
```

### Диалог — копипаст из Telegram

Выдели сообщения в Telegram → Скопировать → Вставить в форму.

Работают оба формата:

**Формат 1:**

```
David Rakhmonov, [26.03.2026 23:49]
Я случайно нет нажал

Миша, [26.03.2026 23:50]
Подожди, а ты пип разве?

David Rakhmonov, [26.03.2026 23:50]
полуфинала ПРОД
```

**Формат 2:**

```
[3/29/26 2:25 PM] Xsenos: На пайчарме такие же проблемы с intelij
[3/29/26 2:25 PM] Xsenos: нет там в логах именно ошибка с джавой
```

Даты убирать не нужно — бот уберёт сам и автоматически определит дату цитаты.

### Картинки

Вставь скриншот прямо в поле цитаты.
Картинка встанет на то место где стоит курсор — как в чате.

Можно комбинировать с текстом:

```
David Rakhmonov, [26.03.2026 23:49]
Посмотрите на это
<сюда вставить картинку>

Миша, [26.03.2026 23:50]
Ты серьёзно?
```

### Описание ситуации

Если без контекста непонятно почему это смешно — заполни поле «Описание».

Пример:

> Решил что он «полуфиналист PROD» и запаниковал что нажал «Нет» на вопрос «вы призёр олимпиады». Он не призёр. У PROD нет полуфинала. «Нет» — правильный ответ.

## 💡 Предложить фичу

[Открой Issue](../../issues/new?template=feature.yml) → опиши идею.

## 🔧 Написать код

1. Нажми **Fork** на странице репозитория
2. Склонируй к себе:
   ```bash
   git clone https://github.com/ТВОЙ_ЛОГИН/prod-shame-board.git
   cd prod-shame-board
   ```
3. Установи зависимости (рекомендуется [Bun](https://bun.sh)):
   ```bash
   bun install
   ```
4. Создай ветку:
   ```bash
   git checkout -b feat/моя-фича
   ```
5. Запусти сервер для разработки:
   ```bash
   bun dev
   ```
   Сайт будет доступен по адресу [http://localhost:4321](http://localhost:4321)
6. Собери проект для проверки:
   ```bash
   bun run build
   ```
7. Коммить и пушь:
   ```bash
   git add .
   git commit -m "feat: описание"
   git push origin feat/моя-фича
   ```
8. Открой Pull Request

## Правила

- ✅ Только реальные цитаты из чата
